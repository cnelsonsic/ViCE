Architecture¶

As you can see from the architecture stack diagram above, ViCE’s core is composed of the plugin framework and database layer [1]. Item, Action, and other, yet to be developed plugin types sit directly on these components. While not yet started, other features and abstractions suitable for designers and players will then be built on top of these.

Note

The opacity of each box represents an approximation of how complete the represented component is.

Plugin Framework¶

ViCE’s plugin framework is based on the simpole fact that all new-style classes [2] in python know about their subclasses. All plugins [3] must be given a name by assigning the class attribute NAME. It is through this name that plugins are identified, and without it, a plugin won’t be discoverable.
